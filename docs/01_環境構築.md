# 環境構築ガイド

本プロジェクトは **GitHub Codespaces** 上での開発を前提としています。
個人のPCにJavaやMavenをインストールする必要はなく、ブラウザとGitHubアカウントがあれば誰でも同じ環境で開発を始めることができます。

## 1. Codespacesの起動
1.  リポジトリのトップページで、`< > Code` ボタンをクリックします。
2.  `Codespaces` タブを選択し、`+` ボタン（Create codespace on main）をクリックします。
3.  バックグラウンドで開発環境の構築が始まり、数分待つとブラウザ上でVS Codeが起動します。

## 2. 開発環境の構成
`.devcontainer/devcontainer.json` ファイルによって、以下のツールが自動的にセットアップされます。

-   **Java**: バージョン21
-   **Maven**: Spring Bootが推奨するバージョン
-   **VS Code拡張機能**:
    -   `vscjava.vscode-java-pack`: Java開発に必要な拡張機能群
    -   `vmware.vscode-spring-boot`: Spring Boot開発を支援するツール
    -   `vscjava.vscode-maven`: Mavenプロジェクト管理を支援
    -   `humao.rest-client`: HTTPリクエストをテストするクライアント

## 3. 動作確認
-   Codespacesの起動が完了すると、`postCreateCommand` で定義された `java -version` と `mvn -version` が実行され、ターミナルにそれぞれのバージョン情報が表示されます。
-   これにより、開発環境が正しく構築されたことを確認できます。
