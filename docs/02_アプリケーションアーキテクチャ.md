# アプリケーションアーキテクチャ

本プロジェクトは、Spring Bootで一般的に用いられる **3層アーキテクチャ（3-Tier Architecture）** を採用しています。
これにより、各クラスの責務が明確になり、保守性やテストのしやすさが向上します。

## 1. アーキテクチャ図

```
[ブラウザ] <---(HTTP)---> [Controller] <---> [Service] <---> [Repository] <---> [データベース]
   ↑                                                                             (H2 / PostgreSQL)
   |                                                                                   ↑
(HTML/Thymeleaf)                                                                       |
   |                                                                                   |
   +-------------------- [Entity] -----------------------------------------------------+
```

## 2. 各層の役割

### ① Controller (コントローラ層)
-   **役割**: ブラウザからのHTTPリクエストを受け付け、レスポンスを返す窓口。
-   **責務**:
    -   リクエストパラメータの受け取り。
    -   入力値の簡単なチェック（Validation）。
    -   ビジネスロジックを持つ **Service層** のメソッドを呼び出す。
    -   Service層から受け取った結果を `Model` に格納し、表示する画面（Thymeleafテンプレート）の名前を返す。
-   **該当クラス**: `AssetController.java`, `LoanController.java`

### ② Service (サービス層)
-   **役割**: アプリケーションのビジネスロジック（業務処理）を担当する。
-   **責務**:
    -   複数の **Repository** を組み合わせて、複雑なデータ操作や業務ルールを実現する。
    -   トランザクション管理を行う（例：貸出処理では、資産テーブルの更新と貸出テーブルの登録を必ずセットで行う）。
-   **該当クラス**: `LoanService.java`

### ③ Repository (リポジトリ層)
-   **役割**: データベースへのアクセスを担当する。
-   **責務**:
    -   Spring Data JPA の `JpaRepository` を継承することで、基本的なCRUD（Create, Read, Update, Delete）操作のメソッドが自動的に提供される。
    -   複雑な検索条件が必要な場合は、メソッド名でクエリを定義したり、`@Query` アノテーションでJPQL/SQLを記述したりする。
-   **該当クラス**: `AssetRepository.java`, `LoanRepository.java`, `UserRepository.java`

### ④ Entity (エンティティ)
-   **役割**: データベースのテーブル構造をJavaオブジェクトとして表現する。
-   **責務**:
    -   `@Entity` アノテーションを付け、テーブルのカラムとクラスのフィールドをマッピングする。
-   **該当クラス**: `Asset.java`, `Loan.java`, `User.java`

## 3. リクエストの流れ（例：GET /assets）
1.  ブラウザが `GET /assets` をリクエストする。
2.  `AssetController` の `list()` メソッドがリクエストを受け取る。
3.  `list()` メソッドが `AssetRepository` の `findAll()` を呼び出す。
    - (この簡単な例ではService層を省略しています)
4.  `AssetRepository` がデータベースの `assets` テーブルから全件データを取得する。
5.  取得した `Asset` のリストが `AssetController` に返される。
6.  Controllerは、取得したリストを `Model` に `"assets"` という名前で格納する。
7.  Controllerは、`"assets"` という文字列を返し、Thymeleafが `templates/assets.html` を使ってHTMLを生成し、ブラウザに返す。
