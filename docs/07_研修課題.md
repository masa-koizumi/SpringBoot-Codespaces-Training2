# 研修課題カリキュラム

本プロジェクトを使用した段階的な研修課題です。

## ステップ1：初級向け（記憶と理解）

**目標**: Webアプリケーションの基本的なデータフロー（画面→Controller→DB）を理解し、既存の仕組みに合わせて軽微な修正ができる。

### 課題 1-1: データの追加と確認
1.  `src/main/resources/data.sql` を編集し、初期データとして「新しいユーザ」と「新しい資産」を追加してください。
2.  アプリケーションを再起動し、ブラウザおよびH2コンソールでデータが反映されていることを確認してください。

### 課題 1-2: 資産への「価格」項目の追加
資産管理に「価格（price）」情報を追加します。既存の `location`（保管場所）の実装を参考にしてください。
1.  **DB定義変更**: `src/main/resources/schema.sql` の `assets` テーブルに `price INTEGER` カラムを追加。
2.  **初期データ修正**: `src/main/resources/data.sql` のINSERT文に価格データを追加。
3.  **Entity修正**: `com.example.demo.entity.Asset` に `price` フィールドを追加。
4.  **Controller修正**: `AssetController` の登録処理で `price` を受け取り、保存するように修正。
5.  **画面修正**: 資産一覧画面 (`assets.html`) と登録画面 (`asset-new.html`) に価格の表示・入力欄を追加。

## ステップ2：リスキリング向け（実践力）

**目標**: 堅牢なアプリケーションを作成するための「入力チェック」「トランザクション」「例外処理」を正しく実装できる。

### 課題 2-1: 入力チェック（Validation）の導入
不正なデータが登録されないようにバリデーションを追加します。
1.  `pom.xml` に `spring-boot-starter-validation` が含まれていることを確認してください。
2.  **Entity/Formの修正**: `Asset` クラス（または専用のFormクラス）のフィールドにバリデーションアノテーションを付与してください。
    - 資産名：`@NotBlank`, `@Size(max=100)`
    - 価格：`@Min(0)`
3.  **Controllerの修正**: `AssetController` の登録メソッド引数に `@Validated` を付与し、`BindingResult` でエラー判定を行ってください。
4.  **画面の修正**: エラーが発生した場合、登録画面に入力値を保持したまま戻り、エラーメッセージを表示するように実装してください。

### 課題 2-2: 貸出・返却のビジネスロジック実装
Controllerに書かれているロジックをService層に分離し、データの整合性を保証するトランザクション管理を行います。

1.  `com.example.demo.service.LoanService` クラスを実装してください。
2.  **貸出処理 (`loan`)**:
    - メソッドに `@Transactional` を付与。
    - 資産のステータスチェック（既に `LOANED` なら例外をスロー）。
    - `loans` テーブルへの登録（`save`）。
    - `assets` テーブルのステータス更新（`LOANED`）。
3.  **返却処理 (`returnAsset`)**:
    - メソッドに `@Transactional` を付与。
    - `loans` テーブルから該当データを検索。
    - `assets` テーブルのステータスを `AVAILABLE` に戻す。
    - 返却日を本日の日付で更新する（または論理削除を行う）。
4.  **テスト**: 貸出中の資産をさらに貸し出そうとした際、正しくエラーになるか確認してください。
